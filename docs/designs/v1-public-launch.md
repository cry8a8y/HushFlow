## Accepted Scope (added to this plan)
- Session statistics: stats.log (TSV) + `hushflow stats` CLI + daily/cumulative display
- Homebrew tap + formula
- JSON-based theme system + 3-5 built-in community themes (Catppuccin, Dracula, Nord)
- `hushflow wrap -- <cmd>` universal CLI layer (reuse existing session mechanism)
- CONTRIBUTING.md + GitHub Actions CI + CLAUDE.md + repo public
- Optional chime sounds via ffplay/mpv (async playback)
- Streak counter + session summary display
- Auto-theme: OSC 11 terminal background detection

## Key Architecture Decisions
- Stats format: **TSV** (tab-separated) — bash-native, simple, efficient
- Theme format: **JSON** — standard, community-friendly, parsed by jq
- Wrap mode: **Reuse existing session mechanism** (marker file + breathe-compact.sh)
- Test strategy: **Extend smoke-test.sh** — no external framework needed
- Launch order: **Build all features → then go public** — nail the first impression

## Implementation Order
1. Stats engine (TSV write + `hushflow stats` CLI + streak counter)
2. Community themes (JSON loader + 3-5 built-in themes)
3. Auto-theme (OSC 11 detection)
4. `hushflow wrap -- <cmd>` (signal handling + session reuse)
5. Sound integration (ffplay/mpv async)
6. Go public prep (CI + CONTRIBUTING + CLAUDE.md)
7. Homebrew formula + tap
8. Repo → public
