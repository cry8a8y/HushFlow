## Context

HushFlow v2.0 is technically complete (290+ tests, 4 breathing patterns, 6 animations,
8+ themes, sound system, plugin API, multi-tool support). But adoption is low:
2 GitHub stars, 1 fork, 193 views, 1,261 clones. The gap is **discovery and distribution**,
not technology.

## Vision

### 10x Check
HushFlow evolves from "AI wait-time breathing CLI tool" to "AI developer mindfulness
infrastructure":
- Landing page makes it discoverable via search engines
- VS Code / Cursor extension expands TAM 10x beyond CLI users
- Homebrew makes installation zero-friction
- Stats card lets users do marketing for you
- Awesome lists + HN/Reddit kickstart the discovery flywheel
- Guided onboarding solves the retention gap

### Platonic Ideal
Every AI coding tool user knows HushFlow. Searching "breathing while coding" returns
hushflow.dev as the first result. Install via `brew install hushflow` or VS Code
marketplace. Developers share breathing stats cards on Twitter. GitHub 5,000+ stars.
AI tool official docs recommend HushFlow.

## Architecture Decisions

- **Monorepo**: VS Code extension lives in `extension/` subdirectory, sharing themes,
  sounds, and breathing parameter definitions with CLI
- **Shared config**: Extract breathing mode parameters (inhale/exhale/hold times) to
  `exercises.json`, consumed by both Bash and TypeScript
- **VS Code rendering**: WebView sidebar panel with HTML5 Canvas animations
- **Analytics**: GitHub Traffic only — preserves "zero telemetry" brand promise

## Rollout Phases

```
Phase 1 (foundations):     exercises.json extraction + Guided Onboarding
Phase 2 (distribution):   Homebrew Formula + Landing Page
Phase 3 (shareability):   Stats Card + Awesome List PRs + HN/Reddit
Phase 4 (expansion):      VS Code Extension MVP
```

## NOT in scope
- Gamification / Achievements — P3, streak counter sufficient for now
- Sound Packs marketplace — P3, needs download infrastructure
- Export Stats to JSON — P3, TSV sufficient
- 30-day Challenge mode — P3, gamification extension
- Warp / Fig Integration — P3, low market share
- Apple Health / Google Fit — cross-platform too complex
- JetBrains / Neovim plugins — Phase 2/3 after VS Code proves the model

## Dream State Delta

After this plan ships, progress toward 12-month ideal:
- **Discovery:** 0% → 60% (landing page + awesome lists + HN/Reddit)
- **Distribution:** 50% → 80% (+ Homebrew + VS Code marketplace)
- **Retention:** 30% → 60% (+ onboarding wizard + stats card shareability)
- **Community:** 0% → 20% (foundations laid, flywheel not yet spinning)

## Key Risks
- VS Code extension is the largest work item; mitigated by phased rollout
- Landing page could feel generic ("AI slop"); maintain terminal aesthetic brand
- Awesome list PRs may be rejected; multiple lists = diversified risk
